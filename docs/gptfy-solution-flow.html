When an agent checks “Create Knowledge Base” on resolve, KbMatchAgentService is the sole gate — no SOSL, no 3-tier pipeline
Step 1 — KB_Match_Agent RAG semantic search

KbMatchAgentService invokes KB_Match_Agent with Case Subject + Description (500 chars).

Agent returns KB_MATCHES_START…END with up to 3 article titles from FILE_SEARCH, plus optional KB_UPDATE_SCOPE block.

Response HTML is normalized (<br> → newlines, entity decode) before parsing.

Step 2 — SOQL verify Exact + fuzzy

Exact: WHERE Title IN :agentTitles AND PublishStatus IN ('Draft','Online')

Fuzzy: if agent paraphrases titles, keyword LIKE '%word%' on distinctive terms (e.g. “Thumbnail”, “Excel”).

Any confirmed match → Route 3 SKIP — no new article.

Step 3 — CREATE or stop Route 3 / 4

SKIP (Route 3): writes scope to Case.KB_Candidates__c (KB Update Scope). Agent reviews why AI missed the article.

CREATE (Route 4): chains KbCreationService → KB Creation Prompt → KbArticleBuilderAction inserts draft article only. UPDATE instructions ignored.

Sets Case.KB_Article_Link__c and CaseArticle junction on create.

Email fix: Case_Resolution_Email_RTF sends plain text from Case.Description (not HTML Resolution__c) so customers never see raw tags like <p>.
